## 型定義

### 変数　※プリミティブ型

```ts
let name: string = 'sakaki'
let age: number = 33
let isMan: boolean = true
```

### 配列　※プリミティブ型

```ts
const array: string[] = []
```

### オブジェクト

```ts
const user: {firstName: string, lastName?: string, age: number}
```
※プロパティ?を使用してオプショナル（省略可能）なプロパティとして指定出来る。

### 関数

引数と戻り値について型を指定出来る。オプショナルな引数も指定可

```ts
function sayHello(name: string): string {
  return `Hello ${name}`
}
```

引数に関数をとることも出来る

```ts
function printName(firstName: string, formatter: (name: string) => string) {
  console.log(formatter(firstName))
}

function formatName(name: string): string {
  return ` ${name} san`
}

printName('sakaki', formatName)

```

アロー関数

```ts
const sayHello = (name: string): string => `Hello ${name}`
```

関数の型を指定

```ts
function getBirdsInfo(name: string): string[] {
  return name.split(',')
}

// 引数のbirdInfoは引数がstringで戻り値がstring[]の関数に限定される
function singBirds(birdInfo: (x: string) => string[]): string {
  return birdInfo('hato, kiji')[0] + 'piyo piyo'
}

console.log(singBirds(getBirdsInfo))
```

### 型アサーション

型推論により、変数の初期化を行うと自動的に型が決定される。

型を型推論でなく、明示的に指定したいときに型アサーションを使用する。

型推論ではエラーになる例

```ts
var foo = {};
foo.bar = 123; // Error: property 'bar' does not exist on `{}`
foo.bas = 'hello'; // Error: property 'bas' does not exist on `{}`
```

型アサーションを使用

```ts
interface Foo {
    bar: number;
    bas: string;
}
var foo = {} as Foo;
foo.bar = 123;
foo.bas = 'hello';

```

### 型エイリアス

typeキーワードで指定する。

``` ts
type Name = string

type Point = {
  x: number;
  y: number;
}

function printPoint(point: Point) {
  ...
}
```

関数も型エイリアスで定義する

```ts

type Formatter = (a: string) => string

function printName(firstName: string, formatter: Formatter) {
  console.log(formatter(firstName))
}

```

型エイリアスのtypeのキーも明記しないで定義する

```ts
type Label = {
  [key: string]: string
}

# 型が一致するので正常に動作
const labels: Label = {
  topTitle: 'トップのタイトル',
  topSubTitle: 'トップのサブタイトル'
}

# エラー
const hoge: Label = {
  message: 100
}

```

### インターフェイス

型エイリアスと同じような機能だが拡張可能

```ts
interface Point {
  x: number;
  y: number;
}

function printPoint(point: Point) {
  console.log(`x座標は${point.x}です。`)
  console.log(`y座標は${point.y}です。`)
}

interface Point {
  z: number;
}

printPoint({x: 10, y: 100, z: 1000})
```

implementsを使用してclassに移譲することが可能

```ts
interface Point {
  x: number;
  y: number;
  z?: number;
}

class MyPoint implements Point {
  x: number;
  y: number;
}

```

extendsを使って複数のインターフェイスを継承することも出来る

```ts
interface Colorful {
  color: string;
}

interface Circle {
  radius: number;
}

interface ColorfulCircle extends Colorful, Circle {}

const cc: ColorfulCircle = {
  color:'red',
  radius: 10,
}
```

### ジェネリック型

ジェネリックとはクラス・関数において、その中で使う型を抽象化し外部から具体的な型を指定できる機能

```ts
// Tはクラス内で利用する仮の型の名前
class Queue<T> {
  private array: T[] = []

  getArray() {
    return this.array
  }

  push(item: T) {
    this.array.push(item)
  }

  pop(): T | undefined {
    return this.array.shift()
  }
}

const queue = new Queue<number>()

queue.push(1)
queue.push(2)
queue.push(3)
console.log(queue.getArray())
console.log(queue.pop())

const queue2 = new Queue<string>()
queue2.push('sakaki')
queue2.push('yamamoto')
queue2.push('yamada')
console.log(queue2.getArray())
console.log(queue2.pop())

```

### Union型とIntersection型

和集合がUnion、積集合がIntersection

```ts
type Identity = {
  id: number | string;
  name: string;
}

type Contact = {
  name: string;
  email: string;
  phone: string;
}

// Union型(IdentityとContactのどちらかのプロパティを満たす必要がある)
type IdentityOrContact = Identity | Contact

const id: IdentityOrContact = {
  id: 111,
  name: 'sakaki'
}

const contact: IdentityOrContact = {
  name:'sakaki',
  email:'sakaki@gmail.com',
  phone:'0123456789'
}

// Intersection型(IdentityとContactの両方のプロパティを満たす必要がある)
type Employee = Identity & Contact

const employee: Employee = {
  id: 111,
  name:'sakaki',
  email:'sakaki@gmail.com',
  phone:'0123456789'
}
```
